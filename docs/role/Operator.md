# 진행자 (Operator) 기능 가이드

> 진행자는 배정받은 품목의 구매자(리뷰어) 정보를 관리하고, 이미지 업로드 링크를 공유합니다.

`/operator` 는 **단일 라우트**이며, 모든 화면은 **OperatorLayout 내부의 시트 탭 + 좌측 사이드바**로 처리됩니다.

---

## 1. 로그인 후 첫 화면

### 1.1 화면 구성
- **좌측 사이드바(OperatorSidebar)**: 배정받은 연월브랜드 → 캠페인 계층 구조
- **우측 메인**: 시트 영역 (내부 탭 0번에서 5번)
- **상단 헤더**: 메모장, 프로필

### 1.2 내부 시트 탭 (6개)
시트 영역 상단에는 6개의 탭이 있습니다.

| 탭(sheetTab) | 탭 이름 | 컴포넌트 | 설명 |
|-----|-----|-----|-----|
| 0 | 캠페인 시트 | OperatorItemSheet | 구매자 CRUD (기본 진입) |
| 1 | 날짜별 작업 | DailyWorkSheet | 특정 날짜의 전체 캠페인 구매자 |
| 2 | 기한 내 미제출 | (overdue) | 14일 경과 + 리뷰샷 미제출건 |
| 3 | 월별 조회 | OperatorMonthlyCalendar | 월별 캘린더 (일자별 카운트) |
| 4 | 리뷰샷 검색 | OperatorReviewSearch | 공통 ReviewSearchDashboard (본인 배정 brand 스코프) |
| 5 | 구매자 분석 | OperatorBuyerAnalytics | 공통 BuyerAnalyticsDashboard (본인 배정 한정) |

> 캘린더(탭 3)에서 날짜를 클릭하면 **날짜별 작업 탭(탭 1)** 으로 연동되어 해당 날짜 데이터가 표시됩니다.

### 1.3 사이드바 표시 정보
- 연월브랜드명 (예: "2601오브제")
- 캠페인별 진행률 표시:
  - **100% 완료**: 녹색 체크 아이콘
  - **80% 이상**: 녹색 배경 퍼센트 칩
  - **50% 이상 79% 이하**: 노랑 배경 퍼센트 칩
  - **50% 미만**: 주황 배경 퍼센트 칩
- 신규 배정 품목: 주황색 "신규(NEW)" 칩
- 구매자 없음 경고: 빨강 경고 아이콘
- 캠페인 자연정렬, 새 캠페인/새 품목 실시간 감지 배너

### 1.4 배정 상태 구분
- **당일 배정** = "신규(NEW)" 표시
- **다음날 이후** = "진행" 표시 + 구매자 없으면 경고 아이콘

---

## 2. 메모장 기능 (헤더 버튼)

- 헤더 "메모장"(OperatorMemoDialog) → 개인 메모 작성/조회
- 다이얼로그 닫기 시 자동 저장, 다음 로그인 시 복원

---

## 3. 선 업로드(임시 구매자) 병합

- 구매자 등록 전 이미지가 먼저 업로드되면 임시 구매자(`is_temporary=true`)로 보관됨
- 진행자가 실제 구매자를 등록하면 계좌 정규화 키로 임시 구매자를 찾아 이미지 이관 후 병합
- 시트에서는 노란색 배경 + "선 업로드" 칩으로 구분 (아래 8.2 참고)
- 참고: 과거의 헤더 알림 아이콘(선 업로드 개수 배지, 30초 갱신)은 현재 코드에 없음

---

## 4. 캠페인 시트 (OperatorItemSheet, 탭 0)

### 4.1 컬럼 순서 (제품 테이블)
| 순서 | 컬럼명 | 설명 |
|-----|------|-----|
| 1 | 접기 | 그룹 접기/펼치기 버튼 |
| 2 | 날짜 | 배정된 날짜 |
| 3 | 순번 | 일차별 순번 |
| 4 | 제품명 | 품목 이름 |
| 5 | 옵션 | 구매 옵션 |
| 6 | 플랫폼 | 쿠팡/네이버 등 |
| 7 | 출고 | 미출고/실출고 |
| 8 | 키워드 | 희망 유입 키워드 |
| 9 | 가격 | 상품 가격 |
| 10 | 총건수 | 총 구매 건수 |
| 11 | 일건수 | 일 구매 건수 |
| 12 | 택배 | 택배대행 여부 |
| 13 | URL | 상품 URL |
| 14 | 특이사항 | 리뷰가이드 등 통합 |

### 4.2 시트 기능
- 그룹 접기/펼치기 (접기 컬럼 클릭)
- 컬럼 헤더 필터링, 드래그 복사, Ctrl+C / Ctrl+V
- Ctrl+S 또는 저장 버튼으로 저장
- 컬럼 너비 조정 (자동 저장), Shift+마우스휠 = 횡스크롤

### 4.3 건수/금액 합계
- 필터링/접기 상태와 무관하게 전체 데이터 기준 합계를 시트 하단에 표시

---

## 5. 구매자 추가

### 5.1 단일 추가
- 시트 빈 행에 직접 입력. 컬럼: 주문번호, 구매자, 수취인, 아이디, 연락처, 주소, 계좌번호, 금액

> 데이터는 TEXT 타입으로 유연하게 입력되며 빈 행도 허용됩니다(구매자 수에서 제외). 업로드 매칭을 위해 보통 주문번호 또는 계좌번호 중 하나를 입력합니다.

### 5.2 일괄 추가 (슬래시 파싱)
- 주문번호 컬럼에 슬래시 구분 데이터 붙여넣기 → 자동으로 8개 컬럼에 분배
- 형식: `주문번호/구매자/수취인/아이디/연락처/주소/계좌번호/금액`

### 5.3 슬래시 파싱 예시
```
8100156654664/김민형/김민형/p4che@naver.com/010-8221-1864/경남 거제시 장승포동 123/부산112-2323-738601 김민지/22800
```

### 5.4 다중 행 일괄 추가
- 여러 줄 데이터 한번에 붙여넣기 (각 줄 = 한 구매자)

---

## 6. 구매자 수정 / 삭제

### 6.1 수정
- 셀 클릭 후 수정, Ctrl+S 또는 저장 버튼

### 6.2 삭제
- 행 선택 후 삭제. 연결된 이미지가 있어도 삭제 가능
- 삭제된 구매자의 이미지는 "선 업로드" 상태로 변경

---

## 7. 업로드 링크 공유

### 7.1 링크 복사
- 품목/슬롯 행 선택 또는 우클릭 → "업로드 링크 복사"

### 7.2 링크 형식
- URL: `https://your-domain.com/upload-slot/{토큰}` (슬롯별 고유 토큰)

### 7.3 사용 방법
- 구매자가 링크 접속 → 이름 검색 → 본인 주문 선택 → 이미지 업로드
- 선택한 주문의 buyer_id로 직접 매칭

---

## 8. 이미지 확인

### 8.1 리뷰샷 컬럼
- 업로드 완료 시 썸네일 표시, 클릭 시 확대 팝업

### 8.2 선 업로드 표시
- 구매자 등록 전 이미지 업로드된 경우 노란색 배경 + "선 업로드" 칩
- 구매자 등록 시 자동 매칭(병합)

---

## 9. 입금명 수정

- 구매자 시트의 "입금명" 컬럼만 직접 편집 가능 (입금확인 토글은 Admin만)

---

## 10. 그룹 관리

### 10.1 일차별 그룹
- 일 구매건수에 따라 자동 그룹(day_group) 생성 (예: "6/6" → 1일차 6건 + 2일차 6건)

### 10.2 접기/펼치기 / 삭제
- 접기 버튼으로 그룹 숨김/펼침
- 그룹 전체 삭제 시 하위 구매자도 삭제 (경고 확인)

---

## 11. 일 마감 (day_group 분할)

- 현재 day_group을 마감하면 다음 day_group 슬롯이 생성됩니다.
- 일 마감 시 **진행자 자동 배정** + **현재 day_group의 제품 정보가 새 슬롯에 복사**됩니다.
- 이후 day_group별 제품 정보는 완전 독립 (수정해도 다른 day_group에 영향 없음).

---

## 12. 기한 내 미제출 (탭 2)

- 정보 입력 후 14일이 경과했으나 리뷰샷이 미제출된 슬롯을 모아 표시
- 미제출 건을 빠르게 추적/독려하는 용도

---

## 13. 월별 조회 (OperatorMonthlyCalendar, 탭 3)

- 월별 캘린더에 일자별 카운트(전체/작성/리뷰샷) 표시
- 날짜 클릭 → 날짜별 작업 탭(탭 1)으로 이동하여 해당 날짜 데이터 조회

---

## 14. 리뷰샷 검색 / 구매자 분석 (탭 4 / 탭 5)

- **리뷰샷 검색(탭 4)**: 공통 ReviewSearchDashboard. 본인 배정 브랜드 스코프 내에서 제품명/기간으로 리뷰샷 검색
- **구매자 분석(탭 5)**: 공통 BuyerAnalyticsDashboard. 계좌 단위 구매자 통계 (본인 배정 한정)

---

## 15. 중복 주문번호 확인

- 동일 주문번호 입력 시 빨간색 배경 자동 하이라이팅 (데이터 정합성 확인용)

---

## 16. 숨기기 기능

- 캠페인/연월브랜드 우측 메뉴 → "숨기기" (데이터 유지)
- 사이드바 상단 "숨겨진 항목 보기" 토글로 표시(회색)

---

## 17. 프로필 관리

- 프로필 수정 (이름/비밀번호 변경), 로그아웃

---

## 18. Admin 모드 (Admin이 대신 조회 시)

- Admin이 진행자 대시보드 조회 시 상단에 "Admin으로 돌아가기" 버튼 표시
- Admin이 진행자 대신 구매자 추가/수정/삭제, 업로드 링크 복사 가능

---

## 19. serial(일련번호) 개념

- 브랜드사 계정에는 고유 일련번호(serial, BR0001 형식)가 부여되어 있으며, 진행자 화면에서 동일 이름 브랜드를 구분할 때 활용됩니다.
