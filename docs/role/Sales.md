# 영업사 (Sales) 기능 가이드

> 영업사는 브랜드를 등록하고, 캠페인과 품목을 생성하여 관리하며, 현황 대시보드와 정산을 조회합니다.

`/sales` 는 **단일 라우트**이며, 모든 화면은 **SalesLayout 내부의 상단 탭 + 좌측 사이드바**로 처리됩니다.

---

## 1. 로그인 후 첫 화면

### 1.1 메인 상단 탭 (AppBar 내부)
영업사 화면은 상단 탭 3개로 구성됩니다.

| 탭 | 내부값(sheetTab) | 컴포넌트 | 설명 |
|-----|-----|-----|-----|
| **현황 대시보드** | 2 (기본 진입) | SalesDashboard | 로그인 후 **첫 화면** |
| **캠페인 시트** | 0 / 1 | SalesItemSheet + DailyWorkSheet | 내부 서브탭 [캠페인 시트 / 날짜별 작업] |
| **브랜드 정산** | 3 | SalesBrandSettlement | 본인 캠페인 정산 |

> 첫 진입 기본 탭은 **현황 대시보드**(sheetTab=2)입니다. 새로고침 시에는 localStorage에 저장된 탭 위치가 복원됩니다.

### 1.2 헤더 구성
- 브랜드 추가 (SalesBrandCreateDialog)
- 연월브랜드 추가 (SalesMonthlyBrandDialog)
- 캠페인 추가/수정 (SalesAddCampaignDialog)
- 품목 추가/수정 (SalesAddItemDialog)
- 프로필 / 로그아웃

> 과거 문서에 있던 별도 **"마진 현황" 헤더 버튼은 현재 존재하지 않습니다.** 매출/완료율 등 지표는 현황 대시보드로 통합되었습니다.

### 1.3 좌측 사이드바 (캠페인 시트 / 날짜별 작업 탭에서만 노출)
- 연월브랜드 → 캠페인 계층 구조
- **드래그 정렬** (순서 변경 후 저장)
- **검색** (연월브랜드/캠페인명)
- **정렬** (asc / desc 토글)
- **숨김 토글** (숨겨진 항목 보기/숨기기)
- 캠페인별 **진행률 칩**, 상태 칩, **택배 건수 칩**, **미입력(미입력 날짜) 칩**
- 사이드바 폭 조절 (localStorage 저장)

> 현황 대시보드(sheetTab=2) / 브랜드 정산(sheetTab=3) 탭에서는 사이드바가 숨겨집니다.

## 2. 현황 대시보드 (SalesDashboard)

> BrandDashboard의 CircularGauge / SummaryCard / CampaignSubTable / IssueList 컴포넌트를 재사용하며, `salesDashboardService`로 데이터를 가져옵니다.

### 2.1 구성
- **플랫폼 탭** (맨 왼쪽 "전체" 집계 포함) + 각 탭에 건수/금액 배지
- **요약 카드**: 총 금액 / 구매자 / 리뷰 완료 / 완료율 / 진행 중 캠페인 / 상품 수 등
- **일별 추이 차트** (recharts): 리뷰 완료(LineChart) + 구매자 등록(BarChart), KST 변환 적용
- **이슈 리스트**: 완료율 낮음 / 리뷰 0건 / 금액 상위 TOP
- **제품별 현황 테이블**: 제품명 단위 합산, 정렬/필터/페이지네이션, 행 클릭 시 포함 캠페인 펼침

### 2.2 캠페인 바로가기
- 제품별 현황 → 캠페인 클릭 시 **캠페인 시트 탭(sheetTab=0)** 으로 전환되며 해당 캠페인이 자동 선택되어 시트가 바로 열립니다. (`?openCampaign=:id`)

---

## 3. 브랜드 등록

### 3.1 신규 브랜드 추가
- 헤더 "브랜드 추가" → 브랜드명 입력 → 저장 시 자동으로 본인에게 할당 (BrandSales 매핑)

### 3.2 조건
- 동일 브랜드명 중복 불가
- 본인에게 할당된 브랜드만 사용 가능 (한 브랜드에 여러 영업사 할당 가능)

---

## 4. 연월브랜드 관리

### 4.1 추가
- 헤더 "연월브랜드 추가" → 연월(예: 2601 = 2026년 1월) + 브랜드 선택
- 조건: 동일 연월+브랜드 조합 불가

### 4.2 표시 / 숨기기
- 사이드바에 연월브랜드명 표시 (예: "2601오브제"), 클릭 시 하위 캠페인 펼침
- 우측 메뉴 → "숨기기" / "숨겨진 항목 보기" 토글로 복원

---

## 5. 캠페인 관리

### 5.1 캠페인 추가
- 연월브랜드 선택 후 "+" 또는 우클릭 메뉴
- 입력: 캠페인명(필수), 설명(선택), 시작일/종료일(선택), 상태(신규/보류, 기본값 신규)

### 5.2 캠페인 상태
| 상태 | 색상 | 설명 |
|-----|-----|-----|
| 신규 | 노랑 | 새로 등록된 캠페인 |
| 진행 | 파랑 | 진행자 배정 후 진행 중 |
| 보류 | 회색 | 일시 중단 |
| 완료 | 녹색 | 모든 리뷰 완료 |
| 취소 | 빨강 | 취소된 캠페인 |

### 5.3 수정 / 삭제 / 숨기기
- 캠페인명 우측 편집 아이콘으로 수정
- 삭제: 하위 품목이 없어야 가능 (강제 삭제는 cascade로 휴지통 이동)
- 숨기기: 사이드바에서 숨김 (데이터 유지)

---

## 6. 품목 관리

### 6.1 품목 추가
- 캠페인 선택 후 "품목 추가" 버튼
- 입력 항목:

| 필드 | 설명 |
|-----|-----|
| 제품명 | 제품 이름 |
| 미출고/실출고 | 출고 유형 구분 |
| 희망 유입 키워드 | 검색 키워드 |
| 총 구매 건수 | 전체 구매 목표 건수 |
| 일 구매 건수 | 하루 구매 건수 (슬래시 구분 가능) |
| 상품 URL | 제품 상세 페이지 링크 |
| 구매 옵션 | 구매 시 선택 옵션 |
| 플랫폼 | 쿠팡/네이버 등 |
| 상품 가격 | 단가 |
| 출고 마감 시간 | 당일 출고 기준 시간 |
| 리뷰가이드 | 리뷰 작성 가이드라인 |
| 소구점 | 강조 포인트 |
| 택배대행 | Y/N |
| 비고 | 추가 메모 |

> 대부분의 필드는 TEXT 타입으로 유연하게 입력됩니다. 빈 값 저장도 가능합니다.

### 6.2 일 구매 건수 입력 방식
- 단일 값: `6` (매일 6건)
- 슬래시 구분: `6/6` (1일차 6건, 2일차 6건)
- 다중 일차: `2/2/2/2/2` (5일간 각 2건씩)

### 6.3 품목 수정 / 삭제
- 시트에서 직접 편집 후 Ctrl+S 또는 저장 버튼
- 삭제: 하위 구매자가 없어야 가능, 경고 확인 후 삭제

## 7. 캠페인 시트 (SalesItemSheet, Handsontable)

### 7.1 표시 컬럼
- 날짜, 순번, 제품명, 옵션, 플랫폼
- 출고, 키워드, 가격, 총건수, 일건수, 택배
- 상품URL, 특이사항, 비고

### 7.2 시트 기능
- 드래그 복사, Ctrl+C / Ctrl+V
- 컬럼 너비 조정 (자동 저장), 컬럼 헤더 필터링
- Shift+마우스휠 = 횡스크롤
- 중복 주문번호 빨간색 배경 하이라이팅

### 7.3 날짜별 작업 서브탭 (DailyWorkSheet)
- 캠페인 시트 탭 내부의 두 번째 서브탭
- 특정 날짜의 모든 연월브랜드-캠페인 구매자 데이터를 한 화면에서 조회
- 제품 테이블에 "연월브랜드-캠페인" 컬럼 추가

---

## 8. 구매자 조회 (읽기 전용)

### 8.1 조회
- 품목 선택 시 구매자 정보 표시 (수정/삭제 불가)

### 8.2 조회 컬럼
- 주문번호, 구매자, 수취인, 아이디, 연락처, 주소, 계좌정보, 금액, 입금여부, 리뷰샷

### 8.3 입금명 수정
- 구매자 시트의 "입금명" 컬럼만 직접 편집 가능 (다른 정보는 불가)

---

## 9. 브랜드 정산 (SalesBrandSettlement, 상단 탭)

- 본인 캠페인의 **제품 단위 정산 요약** 조회
- 매출/완료 기준 지표 표시 (입력은 불가, 조회 전용)

> 과거 "마진 현황" 화면은 제거되었으며, 정산 관련 지표는 본 탭과 현황 대시보드로 통합되었습니다.

---

## 10. 프로필 관리

- 프로필 수정 (이름/비밀번호 변경)
- 로그아웃

---

## 11. Admin 모드 (Admin이 대신 조회 시)

### 11.1 표시
- Admin이 영업사 대시보드 조회 시 상단에 "Admin으로 돌아가기" 버튼 표시

### 11.2 기능
- Admin이 영업사 대신 브랜드/연월브랜드/캠페인/품목 생성 가능 (`viewAsUserId`)
- 생성된 항목은 해당 영업사 소유로 저장

---

## 12. serial(일련번호) 개념

- 브랜드사 계정에는 고유 일련번호(serial, BR0001 형식)가 부여되며, 영업사 화면에서 브랜드를 선택하거나 표시할 때 동일 이름 브랜드 구분에 활용됩니다.
