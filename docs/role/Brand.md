# 브랜드사 (Brand) 기능 가이드

> 브랜드사는 자사 캠페인의 리뷰 진행 현황을 모니터링합니다. 계좌정보를 제외한 구매자 컬럼을 조회할 수 있으며, 모든 화면은 읽기 전용입니다.

`/brand` 는 **BrandLayout(자식 라우트 Outlet 구조)** 로 동작하며, 첫 화면은 **현황 대시보드(BrandDashboard)** 입니다.

---

## 0. 실제 라우트 / 상단 탭 구조

### 0.1 라우트 (App.js 기준)
| 라우트 | 컴포넌트 | 설명 |
|-----|-----|-----|
| `/brand` (index) | BrandDashboard | 현황 대시보드 (첫 화면) |
| `/brand/rankings` | BrandRankingView | 올리브영 랭킹 |

### 0.2 상단 탭 (AppBar 내부, 3개)
| 탭 | 라벨 (PC / 모바일) | 화면 |
|-----|-----|-----|
| 현황 대시보드 | 현황 대시보드 / 대시보드 | BrandDashboard (index 라우트) |
| 캠페인 보기 | 캠페인 보기 / 캠페인 | 사이드바 + BrandItemSheet (제한 컬럼) |
| 올리브영 랭킹 | 올리브영 랭킹 / 랭킹 | BrandRankingView (`/brand/rankings`) |

> 첫 진입 기본 화면은 **현황 대시보드**입니다.

---

## 1. 모바일 반응형 / 헤더

### 1.1 모바일 반응형
- `useMediaQuery(theme.breakpoints.down('md'))` 로 모바일 분기
- **캠페인 보기 탭**에서 좌측 사이드바는 모바일에서 **햄버거 버튼 + 임시 Drawer**로 토글
- 탭 라벨이 모바일에서 축약됨 (대시보드 / 캠페인 / 랭킹)

### 1.2 헤더 구성
- **알림 아이콘** (NotificationsIcon)
- **프로필** — 이름 옆에 **serial 표시** (`user.serial` 존재 시 `이름 (serial)` 형식, 예: "오브제 (BR0001)")
- 로그아웃

---

## 2. 현황 대시보드 (BrandDashboard)

> `brandDashboardService`로 데이터를 가져오며, 영업사/관리자 대시보드와 컴포넌트를 공유합니다.

### 2.1 구성
- **플랫폼 탭** (맨 왼쪽 "전체" 집계 포함) + 각 탭에 건수/금액 배지
- **요약 카드**: 총 금액 / 구매자 / 리뷰 완료 / 완료율 / 진행 중 캠페인 / 상품 수 등
- **완료율 게이지** (CircularGauge)
- **일별 추이 차트** (recharts): 리뷰 완료 + 구매자 등록, KST 변환 적용
- **이슈 리스트** (제품명 단위 TOP): 완료율 낮음 / 리뷰 0건 / 금액 상위
- **제품별 현황 테이블**: 제품명 단위 합산, 정렬/필터/페이지네이션, 행 클릭 시 포함 캠페인 펼침

### 2.2 캠페인 바로가기
- 제품별 현황 → 캠페인 클릭 시 `/brand?openCampaign=:id` 로 이동
- BrandLayout이 쿼리를 감지하여 **캠페인 보기 탭**으로 전환 + 해당 캠페인을 자동 선택해 시트를 바로 표시

---

## 3. 캠페인 보기 (BrandItemSheet)

### 3.1 사이드바
- 연결된 연월브랜드 → 캠페인 계층 구조 (PC 고정, 모바일은 햄버거 Drawer)
- 캠페인별 진행률 칩, 상태 칩
- 제품명 통합 검색 툴바
- 숨기기 / "숨겨진 항목 보기" 토글

### 3.2 진행률 계산 기준
- **분모 = ItemSlot 수(슬롯 수)** — 시트의 "전체 N건"과 동일 기준
- **분자 = 리뷰 이미지 업로드 완료(approved) 구매자 수**
- 조건: 선 업로드(임시 구매자)는 제외, 중단(is_suspended) 슬롯 제외

> 과거 문서의 "분모 = 전체 등록 구매자 수"는 잘못된 설명입니다. 현재 구현/정책은 **슬롯 수 기준**으로 통일되어 있으며, 영업사/진행자/브랜드 3개 역할이 동일 기준을 사용합니다.

---

## 4. 리뷰 현황 조회 (캠페인 보기 시트)

### 4.1 조회 가능 컬럼 (BrandItemSheet 기준)
| 컬럼 | 비고 |
|-----|-----|
| 날짜 / 순번 / 제품명 / 옵션 | 슬롯·제품 정보 |
| 주문번호 / 구매자 / 수취인 / 아이디 | |
| 연락처 / 주소 / 금액 | |
| 송장번호 / 리뷰샷 | |

### 4.2 숨겨진 컬럼
- **계좌정보(account_info)** — 브랜드 뷰에서 제외되는 유일한 구매자 컬럼 ([itemSlotController.js](../../backend/src/controllers/itemSlotController.js) `isBrandView` buyerAttributes / [BrandItemSheet.js](../../frontend/src/components/brand/BrandItemSheet.js) `BUYER_HEADER` 기준)

### 4.3 선 업로드 데이터
- 브랜드사에게는 표시되지 않음 (`is_temporary=false`인 정상 등록 구매자만 조회)

---

## 5. 이미지 조회

### 5.1 테이블 뷰
- 리뷰샷 컬럼에 썸네일 표시, 클릭 시 확대 팝업
- 이미지 우클릭 → "이미지 저장" (브라우저 기본 기능)

---

## 6. 올리브영 랭킹 (BrandRankingView, 탭/`/brand/rankings`)

- 자사 제품의 올리브영 BEST 랭킹 추이 조회
- 자사 제품 랭킹 / 종합 변동 / 추이 / 이탈 / 요약 / 인사이트
- 랭킹 수집 트리거 가능 (캐시/lock/rate-limit 자동 처리)

> AdminRankingDashboard의 일부 컴포넌트를 재사용하지만, 데이터는 자사 제품 스코프로 한정됩니다.

---

## 7. 알림 기능 (헤더 아이콘)

- 알림 아이콘 클릭 시 알림 목록 표시
- 알림 종류: 새 캠페인 연결, 리뷰 완료, 기타 시스템 알림
- "모두 읽음" 처리

---

## 8. 프로필 관리

- 프로필 수정 (이름/비밀번호 변경)
- 로그아웃 (헤더 프로필에 serial이 함께 표시됨)

---

## 9. Admin 모드 (Admin이 대신 조회 시)

- Admin이 브랜드사 대시보드 조회 시 상단에 "Admin으로 돌아가기" 버튼 표시
- 브랜드사와 동일한 제한된 뷰 (민감 정보 숨김)
- `view-brand` 라우트는 `/rankings` 자식 라우트를 포함

---

## 10. 제한 사항

### 10.1 조회만 가능
- 구매자 추가/수정/삭제 불가, 캠페인/품목 수정 불가

### 10.2 정보 접근 제한
- 계좌정보(account_info)는 브랜드 뷰에서 제외됨 (그 외 연락처/주소/금액 등은 조회 가능)

### 10.3 업로드 링크 접근 불가
- 업로드 링크 복사 기능 없음

---

## 11. serial(일련번호) 개념

- 브랜드사 계정은 생성 시 고유 일련번호(serial, BR0001 형식)를 부여받습니다.
- 헤더 프로필에 `이름 (serial)` 형태로 표시되어 동일 이름 브랜드를 구분합니다.
